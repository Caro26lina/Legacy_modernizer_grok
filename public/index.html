Context Optimization Pipeline
✓
Parse AST — Build an abstract syntax tree from the legacy source, identifying all functions, procedures, and data divisions.
✓
Dependency extraction — Walk the call graph recursively from the target function. Collect only what it directly or transitively calls.
✓
Dead code elimination — Remove unreachable branches, deprecated paragraphs, commented-out blocks, and obsolete COPY members.
✓
Comment sanitization — Strip misleading / outdated inline comments that contradict the actual logic (a major source of hallucinations).
✓
Token budget enforcement — If optimized context exceeds 60% of the model window, further prune by abstracting leaf utilities to stubs.
✓
Grok API call — Feed only the optimized slice to Grok with explicit type-safety and idiomatic style instructions.
✓
Post-process & validate — Syntax-check the output, flag undefined references, and surface per-function confidence scores.
Why context optimization matters

LLMs hallucinate more with irrelevant context. A 10,000-line COBOL program modernizing one function needs only ~800 lines of its actual dependencies. Feeding the full file increases wrong-code suggestions by 3–5×.
